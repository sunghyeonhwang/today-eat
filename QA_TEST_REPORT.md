## 3. 발견된 버그

### BUG-001: 리뷰 작성 화면 뒤로가기 버튼 오동작 ✅ **해결됨**

**심각도:** Medium
**재현 가능성:** 항상
**해결 날짜:** 2026-01-28
**해결 커밋:** 2616067

**설명:**
리뷰 작성 화면에서 뒤로가기 버튼을 클릭하면 리뷰 목록 화면(#screen-reviews)으로 돌아가지 않고 홈 화면(#screen-home)으로 이동합니다.

**재현 단계:**
1. 하단 네비게이션에서 '리뷰' 탭 클릭
2. '+ 작성' 버튼 클릭
3. 리뷰 작성 화면 진입 확인
4. 왼쪽 상단 '←' (뒤로가기) 버튼 클릭

**예상 결과:**
리뷰 목록 화면(#screen-reviews)으로 이동

**실제 결과 (수정 전):**
홈 화면(#screen-home)으로 이동

**원인:**
- `client.js`의 1283번 라인에서 모든 뒤로가기 버튼이 무조건 `showScreen('home')`을 호출하도록 하드코딩됨
- 현재 화면 컨텍스트를 고려하지 않음

**해결 방법:**
1. `handleBackAction()` 함수 추가
   - 현재 화면(`currentScreen`)에 따라 적절한 이전 화면으로 이동
   - `reviewWrite` 화면: `reviews` 화면으로 이동
   - 나머지 화면들: `home` 화면으로 이동
2. 뒤로가기 버튼 핸들러를 `handleBackAction()` 호출로 변경

**영향 범위:**
- Desktop Chrome (Chromium) ✅ 수정됨
- Mobile Chrome ✅ 수정됨
